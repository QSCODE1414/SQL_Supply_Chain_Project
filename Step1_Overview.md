# SQL_Supply_Chain_Project

Supply Chain Management System (SQL Project)
📌 About This Project

This project is a Supply Chain Management System built using SQL (MySQL).
The main idea is to understand how real-world business data (like customers, orders, products, and suppliers) is stored and managed using a database.

In simple words, this project shows how companies track everything from orders to deliveries using SQL.

🎯 Project Goal

The goal of this project is to build a complete database from scratch and learn how to work with it.

In this project, I have:

Designed a relational database structure
Created tables using SQL
Connected tables using keys (Primary & Foreign Keys)
Inserted data into tables
Solved real-world business problems using SQL queries
🧠 What You Will Learn
How databases work in real life
How different tables are connected
Writing SQL queries for analysis
Using joins, aggregations, and subqueries
Solving business-related problems using data
🗂️ Project Structure

This project is divided into 5 simple steps:

Project Overview
Understanding what the project is about
Dataset & Requirements
Learning about the data and business rules
SQL Code
Creating database, tables, and inserting data
Practice Questions
Solving real-world SQL problems
Solutions Guide
Checking answers and improving query skills
🛠️ Tools Used
MySQL (for database creation and queries)
MySQL Workbench (for running SQL code)
💡 Why This Project Matters

In real companies, supply chain data is very important.
Businesses use it to:

Track orders
Manage inventory
Work with suppliers
Improve delivery performance

This project gives practical experience similar to real industry work, which is very useful for becoming a Data Analyst.
